# Implementation Plan: Display Original Names for Deleted Accounts and Categories

## Summary

Display original account and category names on transaction cards even when those entities are soft-deleted (archived). Users will see deleted entity names with strikethrough styling to visually distinguish them from active entities. This maintains transaction history accuracy and allows users to audit their financial records.

**Approach**: Minimal lookup approach - expose `isArchived` field in GraphQL schema and add `includeArchived` parameter to queries. Frontend looks up archived entities at display time and applies strikethrough styling based on `isArchived` status.
